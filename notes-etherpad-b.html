Exactly, it tries to access the word count, line count etc. as well. That seems to be less ideal if you want to remove files.

Remove numbers from the front:
wc -l *txt | sort -n | head -1 | cat $(sed -e "s/[0-9]*\s/ /g")

Or ignore it:
wc -l *.txt |sort | head -1 | cat $(awk '{print $2}')

Alternatively 

cat `ls *.txt|head -1`
cat $(ls *.txt|head -1)

Will both display the first text file

In case it gets more complicated I recommend awk :)
